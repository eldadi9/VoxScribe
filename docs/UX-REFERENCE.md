# VoxScribe - UX Reference

## Purpose
This document captures UX and UI direction for VoxScribe.

The goal is to define the desired user experience, product flow inspiration, interface principles, and external reference links before implementation begins.

## Product UX Goal
VoxScribe should feel:
- simple
- modern
- fast
- clear
- professional
- AI-powered without feeling complicated

The product should help users go from audio or meeting input to transcript, summary, and export with minimal friction.

## Main UX Principles
- keep the flow simple and easy to understand
- reduce user confusion during upload, recording, and processing
- make status and progress visible
- keep key actions obvious
- support a clean and professional SaaS look
- design for both desktop and mobile responsiveness
- avoid cluttered screens
- avoid too many decisions at once

## Core User Flows

### 1. Upload Flow
The upload flow should feel quick and guided.

Desired steps:
1. user selects or drags a file
2. system confirms file received
3. user selects language or auto detect
4. user selects summary type
5. user starts processing
6. system shows status
7. transcript becomes available
8. summary becomes available
9. user exports or shares result

### 2. Recording Flow
The recording flow should feel focused and lightweight.

Desired steps:
1. user enters recording page
2. user selects language or auto detect
3. user starts recording
4. user sees recording status clearly
5. user stops recording
6. system processes audio
7. transcript and summary are shown
8. user exports result

### 3. Transcript Review Flow
The transcript screen should make content easy to read and use.

Desired features:
- readable text layout
- timestamps
- clear speaker separation when available
- copy action
- export action
- summary generation access
- future search within transcript

### 4. Summary Review Flow
The summary screen should make outputs easy to scan.

Desired features:
- summary type label
- concise section layout
- action items clearly separated
- workflow or next steps clearly visible
- export or share action
- regenerate option in later phase

## UI Direction
The interface should follow these general directions:
- modern SaaS style
- clean navigation
- strong hierarchy
- large clear buttons for main actions
- readable cards and panels
- simple status indicators
- minimal visual noise
- responsive layout

## Suggested Main Screens
- landing or entry screen
- dashboard
- upload screen
- recording screen
- transcript screen
- summary screen
- history screen
- settings screen

## Important UI Areas

### Dashboard
Should clearly highlight the main actions:
- upload file
- start recording
- view recent jobs
- open transcript history
- open summaries

### Upload Area
Should support:
- drag and drop
- file picker
- visible supported formats
- clear progress status
- simple next action

### Processing Status
The system should always show:
- uploaded
- converting
- transcribing
- summarizing
- completed
- failed

### Transcript Viewer
Should support:
- readable line spacing
- speaker grouping
- timestamps if available
- export controls
- simple navigation

### Summary Viewer
Should support:
- summary mode label
- section blocks
- action items formatting
- next steps formatting
- export controls

## Tone and Feel
The product should feel:
- efficient
- trustworthy
- calm
- smart
- modern

Avoid:
- playful or childish design
- overloaded dashboards
- unclear icons
- hidden main actions
- too many advanced options in the first experience
